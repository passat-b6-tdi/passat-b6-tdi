## Engineering Principles

- Security first: every contract should be designed around failure modes.
- Tests are part of the architecture, not an afterthought.
- Explicit state transitions are better than hidden assumptions.
- Gas optimization should never compromise correctness.
- Upgradeability requires strict storage layout discipline.
- Production code should be readable, testable, and audit-friendly.
- Protocol logic should be simple at the interface level and strict internally.
